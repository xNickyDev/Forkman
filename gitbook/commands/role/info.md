# /role info

## Usage

```
/role info <role>
```

## Arguments

| Name | Description                 | Type   | Required |
| :--: | :-------------------------: | :----: | :------: |
| role | The name or ID of the role. | String | Yes      |

{% hint style="info" %}
You need to provide the **exact** role name or ID, both works. The argument `role` is not case sensitive.
{% endhint %}
